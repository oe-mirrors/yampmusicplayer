wichtige Hinweise zu benutzerdefinierten Skins / Important Remarks to Custom Skins
----------------------------------------------------------------------------------
AlfredENeumann, last change 17.03.2024			

Dieses Verzeichnis dient dazu, bei Bedarf einen angepassten, benutzerdefinierten Skin zu erstellen.
Nach einer Neuinstallation ist in dieses Verzeichnis leer, bis auf diese Datei und die Dummy-Versions-Info-Datei V3.x.x.x.ver.

Bei einem Update sollten die Dateien der Vorversion - falls da ein benutzerdefinierter Skin existierte - automatisch hierher kopiert worden sein, wenn es wie vorgesehen funktioniert hat.

Wenn dabei (beim Kopieren)etwas schiefgegangen ist, sollte der alte custom skin VOR einem kompletten Neustart noch in /tmp zu finden sein und kann manuell von dort kopiert werden.

So oder so muss der dann wahrscheinlich noch auf die neue Version angepasst werden, um eventuelle neue Funktionen zu aktivieren; am Besten eben mit dem Original-Skin vergleichen.

Wenn der Skin dann erfolgreich auf die neue Version angepasst wurde, kann und sollte die Datei V3.x.x.x.ver auf die neue Version umbenannt werden (also z.B. V3.3.1.2.ver oder V3.3.2.ver). Dann entfällt die "Anpassung-Notwendig-Warnung" beim Start.

Sollte Yamp nach einer Änderung im benutzerdefinierten Skin nicht mehr starten, weil ein Fehler gemacht wurde: Einfach irgendeine Datei im Skin löschen oder umbenennen, dann startet Yamp mit dem zugehörigen Hauptskin.


Viel Spass
AlfredENeumann

----------------------------------------------
This folder may be used to create your own "custom" skin for Yamp.
After a fresh installation this folder will be empty, except this file and the Dummy-Versions-Info-File V3.x.x.x.ver.

On an update, if there was an existing custom skin in the previous installation, all files of your custom skin should have been automatically copied to here, if everything worked as it it should.

If not (error in automatic copy), you probably will find your previous custom skin BEFORE a complete restart in /tmp and may copy it manually.

In either case, your old custom skin most likely has to be adapted to the new version, at least to activate possible new features. Please compare with the original skin.

After the skin has succesfully updated to the new version, you may and should rename the file V3.x.x.x.ver to the version number of the new version (for example to V3.3.1.2.ver or V3.3.2.ver). Like that, you will not get the Warning "skin update required" any more.

If Yamp does not start anymore after a (faulty) change in the custom skin, just delete or rename any file in thie custom skin, and Yamp will start the next time with the according main skin.

Have fun!
